Partner Onboarding ​

1. Creating a SIMAST Account (Partner Onboarding) ​

To register an application that will interact with the SIMAST API, the organization that owns the application must first be onboarded as a Partner in the MAST information system. This onboarding typically consists of submitting a Partner account request and/or creating the Partner owner account in the MAST system.

The Partner account is not immediately active. It must go through a controlled validation flow:

  • MAST Validation: The Partner account request is reviewed and validated by MAST.
  • SIMAST Activation: Once validated, the SIMAST team activates the Partner account so it can be used to register applications and access the API.

This process ensures that only verified partners can create applications, obtain credentials, and consume SIMAST services under an auditable and approved access model.

Steps (UI Flow) ​

  1. Open the MAST portal and navigate to the Partner onboarding or registration section.

https://sharing.test.simast.info

  1. Submit the Partner account request (or create the Partner owner account if the process is internal).
  1. Wait for MAST validation.
  2. After validation, the SIMAST team activates the account.

2. Registering the Application ​

After your Partner account has been validated by MAST and activated by the SIMAST team, you must complete the initial security step: first login and mandatory password change.

Once your account is fully operational, you can register your application in SIMAST by using the API Manager (French UI label: "Gestionnaire API"). Registering an application is required in order to generate the credentials and configuration needed for API access.

Prerequisites ​

  • Partner account request validated by MAST
  • Partner account activated by the SIMAST team
  • First login completed
  • Initial password changed successfully

Steps (UI Flow) ​

  1. Sign in to the SIMAST portal with your Partner owner account.
  2. Complete the First Login Password Change (if prompted).
  3. From the main navigation, open API Manager ("Gestionnaire API").

  1. Select Register Application (or equivalent action) to create a new application entry.
  2. Fill in the application details (e.g., name, description, callback URL if applicable).

  1. Save to complete the registration.

Validation and Activation Rules (Agreement-Based) ​

If a formal agreement has already been signed between MAST and the Partner during the Partner account creation process, the application will be automatically validated and activated once it is registered.

If no agreement exists at the time of registration, the application will remain pending until one of the following conditions is met:

  • A signed agreement is concluded between MAST and the operating Partner; or
  • MAST grants an exceptional authorization, which is then communicated to the SIMAST team for activation.

In all cases, application activation remains subject to MAST’s governance and SIMAST’s operational enforcement of access controls.

3. Public Keys & Client Secrets ​

SIMAST allows an application to have multiple public keys and client secrets. This enables key rotation, parallel environments (e.g., staging vs. production), and controlled credential lifecycle management.

However, SIMAST reserves the right—at its sole discretion—to revoke any key or secret without prior notice, particularly in cases of suspected misuse, security incidents, policy violations, or governance requirements.

Generating a Public Key and Client Secret ​

To generate the public key and client secret required to obtain an application access token, navigate to the credentials section (Generate Secret - Generer Secret) from the API Manager.

Important Security Note (One-Time Secret Display) ​

The client secret is displayed only once, at the time it is created. After you leave the creation screen, it will not be shown again.

  • Save the client secret immediately.
  • Store it in a secure password manager or secrets vault (recommended).
  • If the secret is lost or exposed, generate a new one and revoke the compromised credential.
