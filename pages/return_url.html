Return URL ​

Request ​

The return URL serves as a inverse communication bridge between SIMAST and third-party applications (requesters). Through it, SIMAST queries the third-party application about the shared data.

Parameters ​

PropertyTypeRequiredDescription
..............
return_urlStringtrueGET return link that the simast will use for feedback.

Return Url Properties ​

The return url properties must be

  • HTTP Method: GET
  • Content Type: application/json

Return Url Response ​

The return URL must follow this format, with two additional fields:

  • beneficiary (Boolean): Indicates whether the individual is a beneficiary (true or false).
  • support_nature (Nullable Text): Describes the nature of the support; this field can be left empty if not applicable.

These fields enable SIMAST to query third-party applications about the shared data.

JSON
{
     data : [
       {
         "id": 0,
         "code": "string",
         "version": 0,
         "departement": 0,
         "commune": 0,     
         ...[Households Fields include in shared request],
         "beneficiary": "boolean",
         "support_nature: "text|null"
         "membres": [
            ...[Members Fields include in shared request],
         ]
     ],
     meta: {
           "current_page": 0,
          "from": 0,
          "per_page": 0,
          "to": 0,
          "total": 0,
          "last_page" : 0
     }      
}

Request example ​

Step 1 ​

Third App make request for data

js
axios.defaults.headers.common['Authorization'] = ACCESS_TOKEN;

axios.post('BASE_URL/v1/third-app/activities/activity:uuid/demandes', 
      {
            "name" : "PRO_2_ACT_1_REQUEST 1.0",
            "description": "Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua. Ut enim ad minim veniam, quis nostrud exercitation ullamco laboris nisi ut aliquip ex ea commodo consequat. Duis aute irure dolor in reprehenderit in voluptate velit esse cillum dolore eu fugiat nulla pariatur. Excepteur sint occaecat cupidatat non proident, sunt in culpa qui officia deserunt mollit anim id est laborum.",
              "quantity": 100
              "fields": [
                    "department", "commune", "section", "nom_chef", "prenom_chef"
              ],
              "filters" : [
                [
                  ["departement", "in", ["8"]],
                  ["commune", "in", ["814"]]
                ]   
              ],
              "return_url": "http://third-app-url/path_request_return_data"
          },
       [
          headers: {
            'Content-type': 'application/json',
          }
      ],
})

Step 2 ​

Third App get data

js
axios.defaults.headers.common['Authorization'] = ACCESS_TOKEN;

axios.get('BASE_URL/v1/third-app/demandes/demande:details:uuid/data', 
       {
          params: {
            current_page: 0,
            per_page: 0
          }
          headers: {
            'Content-type': 'application/json',
          }
      },
})

Step 3 ​

Internally, the third-party application performs several operations, including:

  • Verification of the shared data
  • Updating existing records in it database
  • Distribution of relevant information
  • Other internal processes as required

Step 4 ​

SIMAST queries the shared data through the return URL at the appropriate time, based on its internal processing schedule or operational triggers.

js

axios.get('http://third-app-url/path_request_return_data', 
       {
          params: {
            current_page: 0,
            per_page: 0
          }
          headers: {
            'Content-type': 'application/json',
          }
      },
})
